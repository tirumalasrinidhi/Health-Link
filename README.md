## 🔍 Problem Statement

Modern hospitals face critical operational challenges:

- **Long waiting times** due to manual queue and scheduling management.
- **Error-prone paper-based appointment systems** causing double bookings and missed consultations.
- **Inaccurate manual vital sign recording**, leading to data entry errors and delayed diagnoses.
- **Lack of automated health monitoring**, resulting in slow clinical response to critical conditions.

**HealthLink 2035** addresses these challenges by providing a centralized, web-based Smart Hospital Management System. It enables instant patient registration, real-time appointment scheduling, digital vital sign entry with client-side validation, and an AI-powered Alert Engine that flags abnormal vitals (tachycardia, bradycardia, fever, hypothermia, hypertension, hypotension). The interactive dashboard gives clinicians a 360-degree patient view — demographics, appointments, vitals history, and live health alerts — all searchable by Patient ID.

## 🚀 Installation & Setup

### 1. Install Dependencies

Open your terminal in the `HealthLink2035/` folder and run:

```bash
npm install
```

This will install `express`.

### 2. Start the Server

```bash
node server.js
```

### 3. Open in Browser

Go to: http://localhost:8080

You will be greeted by the **Select Role** landing page (`index.html`).

## 🎯 Usage / Demo Workflow

1. **Register a Patient** → Go to `/PatientRegistration.html`, fill in details, submit.
2. **Schedule an Appointment** → Go to `/AppointmentScheduling.html`, enter Patient ID + Doctor ID + date/time.
3. **Record Vitals** → Go to `/VitalsEntry.html`, enter Patient ID + heart rate + BP + temperature.
4. **View Dashboard** → Go to `/dashboard`, search by Patient ID to see all details + alerts.
5. **Check Doctor Schedule** → Go to `/doctor-schedule`, enter Doctor ID + date.
6. **View Vitals Analytics** → Go to `/vitals-analytics`, enter Patient ID for averages + history.

## 🤖 AI Alert Engine

The AI logic is handled securely within `server.js` and evaluates vital signs against clinical thresholds:

| Condition | Alert Generated |
|-----------|-----------------|
| Heart Rate > 100 | ⚠ **Tachycardia Alert** |
| Heart Rate < 50 | ⚠ **Bradycardia Alert** |
| Temperature > 38°C | 🌡 **Fever Alert** |
| Temperature < 35°C | 🌡 **Hypothermia Alert** |
| Systolic BP > 140 | 💓 **Hypertension Alert** |
| Systolic BP < 90 | 💓 **Hypotension Alert** |
| All within range | ✅ **Vitals Normal** |

Multiple alerts can fire simultaneously (e.g., fever + tachycardia).

## 📊 Analytics & Reporting

Calculations are computed dynamically in `server.js` when routing pages:

### Doctor Schedule Filtering
Appointments are filtered on the fly based on the query parameters (`doctorId` and `schedDate`).

### Vitals Analytics
The backend filters vitals by `patientId` and calculates the aggregated statistics (Average Heart Rate, Average Blood Pressure) using JavaScript array reduction techniques.

## 🏗 Architecture

```
┌─────────────────────────────────────────────────┐
│                   BROWSER                        │
│  HTML Forms ─── CSS Styles ─── JS Validation     │
└──────────────────┬──────────────────────────────┘
                   │ HTTP POST/GET (JSON APIs / HTML)
┌──────────────────▼──────────────────────────────┐
│           NODE.JS EXPRESS SERVER                  │
│                                                   │
│  express.static() ── serves HTML/CSS/JS         │
│                                                   │
│  ┌──────────────────────────────────────────┐    │
│  │        API ROUTES (Controller)            │    │
│  │  /RegisterPatient /ScheduleAppointment    │    │
│  │  /RecordVitals ──► AI Alert Function      │    │
│  │  /api/doctors     /api/patients           │    │
│  └──────────────────────────────────────────┘    │
│  ┌──────────────────────────────────────────┐    │
│  │       PAGE ROUTES (View Engine)           │    │
│  │  /dashboard  /doctor-schedule             │    │
│  │  /vitals-analytics                        │    │
│  └──────────────────────────────────────────┘    │
└──────────────────┬──────────────────────────────┘
                   │ FileSystem (fs)
┌──────────────────▼──────────────────────────────┐
│               JSON DATABASE (Model)               │
│               database/data.json                  │
└─────────────────────────────────────────────────┘
```

**Pattern**: Client-Server API Architecture with Dynamic Page Rendering
- **Model**: JSON File via fs read/write 
- **View**: Static HTML pages + dynamic template rendering in server
- **Controller**: Express Route Handlers
